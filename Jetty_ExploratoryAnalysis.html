Exploratory Data Analysis for the Jetties area in the City of Geelong¶

Business Problem¶

The City of Geelong (CoG) wants to perform a high level exploratory data analysis for different asserts in the Jetties area. The analysis and the forthcoming recommendations will help CoG to improve and seamlessly manage the data management systems.

Description of the Data¶

The CoG provided an extract of the data stored in a Excel file: Jetties.xlsx comprising 28 rows and 40 columns. The columns represent various data attributes, which a yet to be described (TBD), including e.g., the following:

Conclusion and Recommendations¶

The exploratory data analysis highlighted notable data quality issues (inconsistencies) and some initial insights gained from the provided data. The below table lists some of the major issues and the recommendations to mitigate such data quality issues.

Data Quality Issues Description (Comments) Recommendation (Mitigation)
Missing Values Various columns have empty values, zeros or dates in certain/entire records e.g., asset_id, height, depth, width, length, area, _modified Check and fill the records. Else filter out the record if not critical
Duplicates Some columns have Id records assigned all to zeros e.g., pathway_id, polygon_id Check and fill Ids correctly and preserve uniqueness where required
Inconsistent or incomplete columns naming For example, 'field_irri' and created_b Apply clear and consistent columns naming convention
Column Attributes Description Attributes description is missing Provide a concise description of each attribute. This information is crucial when trying to analyse and make sense of the data
Geospatial Data GPS coordinates for assets identified by location names e.g., parkreserv Provide GPS coordinates to overlay some of the features on the map
